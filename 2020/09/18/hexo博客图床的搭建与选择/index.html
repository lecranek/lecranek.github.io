hexo博客图床的搭建与选择

考虑到目前本博客的使用频率和更新质量都不高，暂时不使用任何付费性质的服务进行博客完善。因此，最好的图床方案即是使用一些git托管平台作为图床仓库。

上传工具：PicGo；Picee；MDPIC

图床仓库：Github；Gitee

Markdown编辑器：Typora

Typora + PicGo + Github

官方参考文档：PicGo命令行上传；Typora与PicGo

To use with Typora, we require PicGo ≥ 2.2.0 with PicGo-Server enabled and use port in default value (36677). And when Typora upload images using PicGo, PicGo will be launched and kept running.

参考博客：工具使用篇–typora+picGo实现图片上传；使用 PicGo+GitHub 打造最稳定可靠的免费图床；GitHub+jsDelivr+PicGo搭建快速免费图床

PicGo的安装与操作都相当简便，若使用Typora内置的图片处理功能调用PicGo完成图片上传，将使得上传图片的过程更加无感。

问题

但是这其中仍存在一个问题，由于PicGo本身上传方案的制约，我们只能将所有图片上传到git仓库的单一目录下，因而无法在仓库中对图片进行有效的管理。
另一方面，对于存储于剪贴板中的图片，Typora的处理方式是首先将其写入%APPDATA%/Typora/typora-user-images目录，再使用PicGo上传，这无疑会使得该目录越发臃肿。

为解决这两个问题，最好的方法就是自编写一个脚本，使用Typora提供的cutsom command line上传方式，调用脚本完成上传。考虑到我们从一开始就确定使用Git仓库作为图床，那么使用脚本完成图片上传的过程将格外简单，只需要基于原生的git命令即可。

Typora + 自定义py脚本 + Gitee 实现以文档名称为目录分类管理的图床

参考博客：Hexo博客图床搭建与图片管理；脚本自动化压缩图片+上传PicGo指定目录+jsDelivrCDN

  2. 复制图片进入Typora时将%APPDATA%文件剪切至local git repo。
